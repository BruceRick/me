Allegiant Airlines. iOS Developer on this project. The consumer app for the airline. This app allowed customers to purchase flights and vacation packages, view flight itineraries, download boarding passes and access their user account for the airline. Over the course of many years, this app underwent a few overhauls and redesigns. Code base was originally written in Objective-C and used interface builder and UIKit and was gradually ported to Swift. Originally app used a RESTful backend client but eventually, we implemented a GraphQL client. Cocoapods were used to manage dependencies on iOS. UIKit and Interface Builder were used to construct native UI. Constraints were used extensively as Allegiant wanted to be able to support phones over many different generations.

This project repo was hosted on Stash (now named Bitbucket) and managed through JIRA. Teamcity was used to automate the continuous delivery of the app through build plans. Plans included one that ran automatically when a code review was opened and automatically ran tests to ensure features were not broken or bugs introduced during a code merge. Plans that tested migration between different versions of the app. Plans that was manually triggered to run tests, manage provisioning profiles, take screenshots, create app file and submit them to the app store for testing and release. This project used JIRA to manage tasks and bugs as well as manage and measure project progress. Time and estimates were tracked through JIRA and project health was visually shown through graphs and statistics in JIRA.

Standard Apple localization system was used and eventually manual scripts were used to import and export translations to an excel file for translators. This app initially used a RESTful backend and then converted to using GraphQL. RestKit was used on this project initially and eventually we moved towards a more manual approach using the standard Apple networking libraries and Codable protocol to lower overhead and reduce code. When a redesign of the app was done a GraphQL backend was created and I used the Apollo iOS library to interact with the backend. This app accepted and stored customer payments and various security and encryption was implemented. Portions of this app utilized a mobile version of the Allegiant website and this required sending data between a web view and the native view controller. One of the biggest overhauls of this app was the timeline functionality - the ability for users to sign into their accounts and get a screen that ordered their flights by times with UI card elements. This required a lot of UI work to keep UI in reusable files and display in the proper format and also extensive backend changes to accommodate new data changes to support these card items.

I was a part of this app for many years and was part of all portions of the lifecycle of this app. Designing, estimating and implementing features, deployment and release work, general maintenance cycle after a version was released and finally overhauling the app and upgrading to new technologies. During this project, I flew to Las Vegas to work at the Allegiant Airlines headquarters and go over designs and expectations with the client for upcoming releases. I routinely communicated with the client when technical details were required and also demoed features and took feedback on features that were in development. At times on this project, I was the only iOS developer and had to constantly be very conscious of the importance of this project to Allegiant and also to my company financially.

CBSA division of the Government of Canada. iOS Developer on this project. This app was built in Swift using SwiftUI and we used the Swift Composable Architecture library. The app was an internal app for CBSA agents to track packages being shipped into the country. Agents could use the app to take pictures and scan packages and update details and perform exams and electronically log all information.

We used the Swift Composable Architecture to create a Model View - View Model architecture. This helped keep a wonderfully organized app that was separated by feature with each feature then organized by UI logic and business logic. This architecture also was wonderful at helping us maintain testability by making it easy to create test data and testing mechanisms to test all components of the app.

The government was very proactive in using the best and latest technologies and that gave us the ability to develop this app entirely in SwiftUI. This was my first professional experience with SwiftUI and Combine. I absolutely love both and will advocate using both technologies on any projects I work on in the future. SwiftUI helped us create a reactive app that handled all cases in one place and adapt to changes or events in code in one unified place. It also helped create a wonderfully responsive UI that users would expect of an iOS device. The Swift Package Manager was used in favour of CocoaPods to manage all dependencies.

This app used Microsoft Azure to track features, requests and project details. This app used Amazon Web Services to host the backend services as well as pipelines to deploy the app. Authentication was handled through Amazon's Amplify library and this was included and used to authenticate usernames and passwords.
