I was able to demonstrate my skills and leadership to progress between a Junior Mobile Developer to Intermediate Mobile Developer to finally a Senior Mobile Developer.
While I primarily focused on iOS development, I had plenty of opportunity to work in web and android spaces as well as backend services.

Lixar gave me the opportunity to learn project management through tools such as JIRA, Azure and Amazon Web Services. I was also exposed and eventually gained the knowledge to use and maintain CI/CD services such as Bamboo, Teamcity, Azure Pipelines and Amazon Pipelines.

Some of the projects I worked on included an iPhone app that communicated wirelessly with an automobile through a piece of hardware inserted in the OBD port, an internal iPad app called Flydesk that provided services for pilots flying for Allegiant Airlines, the Allegiant Airlines iOS consumer app, and a yet to be released iOS app for Government of Canada CBSA employees to inspect and review packages electronically. These tasks included organizing certain employee shifts and lunch breaks. I also worked in a section of the store that created keys as well as printed pricing labels for the entire store. Printing labels was a good task to work on my organizational skills. Various departments would request labels that would need to be printed before a sale would take effect. Eventually I went back to school and moved to the after hours staff which included evenings and weekends. While my responsibilites were the same as they had been during the day shift - this move actually had some good lessons as well. Most of the afterhours staff were students such as myself which again helped me plan my future out and decide where I wanted my education to go. Lixar I.T. | Junior iOS Developer | 2013-2015
After school I was referred by one of my professors to the wonderful Lixar I.T. where I interviewed and accepted a role of Junior iOS Developer. This was a huge step in my career and was an incredible experience to get me introduced to the world of professional programming.

One of my first tasks at Lixar was the localization of one of their apps called Delphi Connect. This app communicated with a device installed in your car to display info to an app on your phone. The first task I undertook was localizing all text in the app to display in the language selected by the user. I went through the app and created tokens to refer to text that had various translation files. It looked something like "welcome.text" = "welcome in various languages". This helped me get a good understanding of the app and features before I began more complicated tasks. I became more familiar with the project management software Lixar used called JIRA to track my progress on various tasks. I also became familiar with their source control standards on Git, like release, feature, bugfix branches and the process they used for merging and committing changes.

I continued to work with this app but moved towards working on bug fixes and implementing various pages. I wrote both native Objective-C and small bits of HTML and CSS for this app. This was new for me because on this app they used a framework to render HTML and CSS files within native view controllers. I had to learn the relationship between passing data back ad forth between these template files and native files as well as brush up on my HTML and CSS knowledge. I was successfully able to create settings pages and various smaller feature pages using these HTML files. I also successfully fixed bugs and was able to tag JIRA tickets with the appropriate fix versions and which builds to use.

Delphi Connect eventually branched out and wanted to resell this app to various companies. In order to do this I was a part of the theming team that created a code base to properly theme and install the proper resources for the correct builds. I made use of Xcodes schemes to successfully create different schemes for different vendors that would style the app differently for each vendor and only include the assets required. This required changes in Objective-C to make the code base more generic and work correctly for each vendor.

This was the perfect beginning to my career in tech as it gave me the opportunity to settle in, learn and start in a position I'd never worked in before, but also contribute signifantly to a code base and project. I was really happy with my progression and experiences during this time and was very excited when I was promoted to Intermediate Developer.
Lixar I.T. | Intermediate Developer | 2015-2020
After my work on Delphi Connect I was promoted to Intermediate Developer. This came with increased responsibilites like planning, designing and implementing new features on Delphi Connect. I began to take a more hands on role with the interaction with the backend layer and api for the data we displayed on the app.

I was a part of the Eco-Drive feature on Delphi Connect as one of my first roles as an Intermediate Developer. I got to work more closely with our senior developers. I began to design features at a higher level and would review my designs with the senior resources before beginning development. I got the opportunity to review senior developer code as well through code reviews as well as have them more thoroughly review mine. On the Eco-Drive feature I learned and managed the REST system this app utilized to download data from the backend service. I used a library called RestKit on iOS to make network requests, retrieve data and store it in data objects. This coupled with native Objective-C knowledge helped me create the Eco-Drive feature which displayed charts and statistics to the user about their driving behaviours. I had to learn about the use of dependancies through CocoaPods on iOS and how to manage them for builds. I learned about the relationship CI/CD tools played in the release cycle and this came with more experience with Git and merging and managing code for releases and builds.

I eventually moved into the Air space where I worked on various apps for airlines with all kinds of uses. I worked on features that included a timeline feature for the Allegiant Airlines consumer app to see trip details in a timeline view, a login system for the Allegiant Airlines Flydesk app for their pilots to review documents, a credit card purchase system for Allegiant Airlines InFlight app used by flight attendants and many more. REST, CoreData and migrations were critical in all of those features and I was responsible for using these to interact with our backend APIs. I also became much knowledgable with CI/CD's such as Bamboo and Teamcity and had to use and maintain these piplines to manage tests, releases and submissions to the app store.

I joined a team responsible for creating an app called CrewPad that was used by an airline called AirEuropa and their flight attendants. For this project we used reactive programming and a library called ReactiveCocoa. This was brand new to me and an entirely new type of programming I had to learn but eventually I created and maintained features like a sync to download and upload data when a connection becomes available, a messaging feature that would post and get messages between flight attendants and a feature to download and review an HTML file distributed by the airline with guidlines.

My final years at Lixar I moved to a newly created team that worked to create an app for CBSA, a division of the government of Canada. When I started on this project Apple released SwiftUI and Combine and we decided to use these technologies on this project. I learned how to use SwiftUI to create very responsive and clean UI architecture in combination with the combine framework. We used an architecture called Swift Composable Architecture to maintain and better organize our code. This helped with testing and gave us tools to properly and thoroughly testing all features of the app.

While on this app I began to branch out and help out the web portion of this project. I used HTML/CSS/Java within the React framework on a user portal version of the app to fix bugs and create features. I created features like the advanced search and reject list pages. I also had to learn some basic Python to create localization scripts that would automatically export and import text for translation purposes.
